Integrations

Connect external services to give your agents real-world capabilities. Integrations are managed through the MCP (Model Context Protocol) bridge.

Google Workspace

NICE supports OAuth 2.0 for any Google account — personal Gmail, Google Workspace, or any domain.

Setup

  1. Go to Security > Integrations
  2. Click Sign in with Google
  3. Review and authorize the requested scopes on the Google consent screen

Once connected, the mcp-gateway automatically refreshes expired OAuth tokens before each tool call. Each connection card in the Integrations view shows an R (read only) or R+W (read and write) badge so you can see at a glance what the agents on that ship can do.

Gmail

Scope: https://www.googleapis.com/auth/gmail.modify

Your agents can read your mail and take actions on it:

  • Search emails by sender, subject, date, or content
  • Read full email content and attachments
  • List and filter by labels
  • Send new emails, reply to existing threads, create drafts

Calendar

Scope: https://www.googleapis.com/auth/calendar

Your agents can read your calendar and modify it:

  • List upcoming events and check availability
  • Read event details (attendees, location, description)
  • Create, update, or delete events

Drive

Scope: https://www.googleapis.com/auth/drive.file

Your agents can work with files the app creates or opens — nothing else on your Drive is visible:

  • Search and read files the app has already touched
  • Create new files and folders
  • Update file contents and upload new versions

Approval mode

Any write tool (send, create, update, delete, upload, reply, etc.) can be gated by a per-spaceship approval mode. In review mode the agent pauses before every side-effecting action and asks you to approve or deny inline. Switch to autonomous to let the agent run uninterrupted. You can change this per ship in the Bridge > Ship Behaviors panel.

MCP Servers

NICE uses the Model Context Protocol (MCP) for all integrations. MCP servers expose tools that agents can call during missions.

Built-in MCP Servers

ServerPurpose
gmail-mcpGmail search, read, labels
calendar-mcpCalendar events and availability
drive-mcpDrive search, read, metadata
social-mcpSocial media publishing (Buffer, X, LinkedIn)
browser-proxyWeb browsing for agents

How MCP Works

MCP connections are made at the spaceship level. When you connect Gmail to a spaceship, all agents on that crew inherit the connection. The flow:

  1. Agent needs to use a tool (e.g., search Gmail)
  2. ToolRegistry finds the registered MCP tool
  3. McpBridge routes the call to mcp-gateway
  4. mcp-gateway refreshes OAuth tokens if needed
  5. The MCP server executes the tool and returns results
